# Движение и взаимодействия

Снято с whitewill.ru. «Дорогой» интерактив - это не эффекты ради эффектов, а
спокойные, предсказуемые отклики: ничего резкого, всё на одном тайминге.

## Тайминг

- **Базовый переход - `0.3s`** (`--dur`). На нём почти всё. Быстрые мелочи - `0.2s`
  (`--dur-fast`), крупные/появления - `0.5s` (`--dur-slow`).
- **Фирменный easing - `cubic-bezier(0.65, 0.05, 0.36, 1)`** (`--ease-lux`).
  Резкий старт, мягкая посадка - ощущение веса и дороговизны. Применять везде,
  где есть движение.

## Ховеры (микровзаимодействия)

| Элемент | Что происходит | Класс/правило |
| --- | --- | --- |
| Кнопка-солид | бронза → глубже (`--c-accent-hover`) | `.btn--solid:hover` |
| Кнопка-призрак | заливается акцентом, текст чёрный, бордер загорается | `.btn--ghost:hover` |
| Кнопка-стрелка | стрелка начинает «дышать» (arrow-move) | `.btn--arrow:hover .arr` |
| Ссылка-стрелка | текст → акцент, черта удлиняется, gap растёт | `.link:hover` |
| Фото ч/б | становится цветным + лёгкий зум `scale(1.02)` | `.figure--reveal:hover` |
| Карточка | фон чуть приподнимается (графит) | `.card:hover` |
| Плитка категории | подсветка + текст в акцент | `.catbar__item:hover` |

Принцип White Will: **на ховере элемент наливается акцентным цветом**, а не
просто тускнеет. Бронза (или цвет активной темы) - «загорается».

## Появление при скролле

Направленный fade: элемент всплывает на `~1.4em` снизу с прозрачностью
(`fadeUp`), боковые блоки - по диагонали (`fadeRightUp`). Середина анимации -
уже полная непрозрачность (мягкий вход).

- Вешаем класс `.reveal` (или `.reveal--side`), JS добавляет `.is-in` при входе
  в экран. Каскад через `style="--d:.1s"` - задержка для последовательного входа.
- Уважает `prefers-reduced-motion` - при включённом всё показывается сразу.

Сниппет наблюдателя - в [[../web/components|components]].

## Прочее движение (из библиотеки WW)

- `dot-pulse` - пульсация точки (метки, «онлайн», маркеры на карте).
- `arrow-move` - дыхание стрелки в кнопке/ссылке.
- Стрелки-дропдауны - поворот `180deg`, `transition: transform .3s`.
- Ротация слов в заголовке (`rotateWords`) - если нужен живой подзаголовок.

## Чего не делать

- Разные тайминги и easing на соседних элементах - ломает ощущение целостности.
- Резкие/пружинящие/долгие (>0.6s для ховера) анимации - удешевляют.
- Параллакс и «вау-эффекты» без смысла. Движение служит вниманию, не развлекает.
