# Веб-компоненты

Готовые блоки в `web/base.css`. Каркас целиком - `web/starter.html`
(hero + метод + кейс + цифры). Ниже - что есть и как собирать.

## Подключение

```html
<link rel="stylesheet" href="core/tokens.css">  <!-- сначала токены -->
<link rel="stylesheet" href="web/base.css">      <!-- потом компоненты -->
```

## Блоки

| Компонент | Класс | Назначение |
| --- | --- | --- |
| Навбар | `.nav` / `.hero__nav` | монограмма + имя слева, контакты справа |
| Логотип | `.logo`, `.logo__mono`, `.logo__name`, `.logo__tag` | вордмарк + монограмма АК |
| Hero | `.hero`, `.hero__bg/__title/__lede` | фулл-блид фото + крупный заголовок |
| Печать | `.seal` (+ SVG) | бронзовый медальон |
| Надзаголовок | `.eyebrow` | «02 — Метод» с бронзовой чертой |
| Заголовки | `.display` (h1/h2), `h3` | Montserrat прописными |
| Лид/текст | `.lede`, `.muted`, `.faint` | вводный и вторичный текст |
| Кнопки | `.btn--solid`, `.btn--ghost`, `.link` | действия |
| Категории | `.catbar`, `.catbar__item` | панель услуг внизу hero |
| Шаги | `.steps`, `.step`, `.step__num` | нумерованный метод в 3 колонки |
| Кейс | `.case`, `.case__photo/__quote/__by` | ч/б фото + цитата клиента |
| Цифры | `.stats`, `.stat`, `.stat__n/__t` | полоса метрик с линиями |
| Секция | `.section`, `.divider` | обёртка с ритмом и пределом ширины |

## Изображения в блоках (не только фото)

Картинку подбираем под смысл блока - см. [[../core/imagery|доктрину изображений]].
Не ставим портрет в каждый блок. Компоненты:
- `.figure` / `.figure--bw` - фото-область (цвет / ч/б для кейсов).
- `.cutout` / `.cutout--glow` - вырезанный объект или рендер (статуя, предмет).
- `.motif` - контейнер под абстрактный SVG-визуал, когда фото не нужно.
- `.split` / `.split--narrow` - блок «текст + визуал» (визуал любого типа).
Пример блока с SVG-рендером вместо фото - `web/site-render.html`.

## Правила сборки

- Секции разделяем **волосяными линиями** (`--c-line`), не цветными плашками.
- Кнопка `--solid` (бронза) - одна на экран, главное действие. Остальное -
  `.link` или `.btn--ghost`.
- В цифрах одно число подсвечиваем бронзой через `<i>` внутри `.stat__n` -
  это осмысленный акцент на данных, не «украшение».
- **Выделения в заголовках/цитатах - не по умолчанию.** Ровный текст. Когда
  нужно подсветить часть (решает Александр) - класс `.em` (вес) / `.em.is-accent`
  (бронза). Не подсвечиваем куски фраз автоматически «для красоты».
- Цитата клиента - живая, из [[../../business/methodology/cases|кейсов]];
  мат не вычищаем - это голос Александра.
- Адаптив встроен (`@media max-width:860px`): колонки схлопываются в одну.

## Цвет не только на кнопках: плашки, теги, выноски

Чтобы видеть, как цвета взаимодействуют (фоны, плашки, теги), - живой раздел
«Цвет в деле» на витрине и лист `web/colors-in-use.html`.

- **Установщики акцента** `.acc-bronze / .acc-clay / .acc-sage / .acc-eucalyptus /
  .acc-navy / .acc-amber` - меняют `--c-accent` для вложенного и задают `--on-accent`
  (цвет текста ПОВЕРХ цвета). Вешаются на бейдж, выноску или блок.
- **Бейдж/тег** `.badge` - приглушённая плашка (заливка акцентом 15%, текст цветом),
  `.badge--solid` - сплошная заливка с текстом поверх. Точка - `<span class="pip">`.
- **Выноска** `.callout` + `.callout__t` - плашка с акцентной гранью слева и лёгким
  тоном фона. Для «Стратегия / Рост / Внимание / Премиум» и т.п.
- **Цветной разделитель** `.rule-accent` - короткая полоса акцентом.

```html
<span class="badge acc-sage"><span class="pip"></span>Рост</span>
<div class="callout acc-eucalyptus"><div class="callout__t">Стратегия</div><p class="muted">…</p></div>
<section class="theme-navy"> … блок целиком в синей теме … </section>
```

Правило текста на цвете: на залитой плашке текст - **тёмная версия того же
семейства** или `--on-accent`, не чёрный/серый в ноль (см. [[../core/спецификация|спецификацию]]).

## Кнопки, ссылки, формы, движение

Полная доктрина - [[../core/motion|движение]]. В разметке:

```html
<!-- солид (главное действие, бронза) -->
<a class="btn btn--solid">Оставить заявку</a>
<!-- призрак: на ховере заливается акцентом -->
<a class="btn btn--ghost">Подробнее</a>
<!-- кнопка-стрелка: текст слева, стрелка дышит на ховере -->
<a class="btn btn--solid btn--arrow">Обсудить сессию <span class="arr">›</span></a>
<!-- ссылка-стрелка -->
<a class="link">Как проходит</a>
```

Форма:

```html
<label class="field">
  <span class="field__label">Имя</span>
  <input class="input" placeholder="Александр">
</label>
<label class="field">
  <span class="field__label">Сообщение</span>
  <textarea class="textarea" placeholder="Расскажите про команду и задачу"></textarea>
</label>
<button class="btn btn--solid" style="width:100%">Отправить</button>
```

Появление при скролле - класс `.reveal` (или `.reveal--side`), JS включает `.is-in`:

```html
<div class="reveal" style="--d:.1s">…</div>
<script>
  const io = new IntersectionObserver((es) => es.forEach(e => {
    if (e.isIntersecting) { e.target.classList.add('is-in'); io.unobserve(e.target); }
  }), { threshold: 0.18 });
  document.querySelectorAll('.reveal').forEach(el => io.observe(el));
</script>
```

Цветовая тема секции - класс-обёртка (см. [[../core/color|цвет и темы]]):

```html
<section class="theme-emerald"> … кнопки/ссылки/акценты станут эвкалиптовыми … </section>
```

## Связь с приложением konstruktor

`konstruktor/` сейчас в светлом «редакторском премиуме». Это другой регистр.
Если решим привести его к этой системе - переносим палитру в тёмный, заголовки
на Montserrat прописными, акцент на бронзу. Отдельной задачей, не ломая текущее.
